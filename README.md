# 输入法锁定（InputSourceLock）

一个极简的 macOS 菜单栏小工具：当系统输入法被切到 `ABC`（或其它输入源）时，自动切回你指定的目标输入法（例如「微信输入法」）。除了监听系统输入法变更通知之外，应用还会定时检查当前输入法，防止被其他应用突然切走后长时间切不回来。

## 重要限制

- macOS 在**密码框/安全输入（Secure Input）**场景下可能会禁止第三方输入法工作；本工具会在检测到 Secure Input 时**暂停强制切换**，避免反复抢输入法。
- 仍然可能存在系统/应用层面强制切换输入源的情况（例如某些 App 自行调用系统 API 切换），本工具只能在变更发生后再切回。

## 构建

在本仓库根目录执行：

```bash
./build.sh
```

生成产物：`build/输入法锁定.app`（默认尽量构建为 universal2：Intel + Apple Silicon）

## 运行

```bash
open "build/输入法锁定.app"
```

首次运行后在菜单栏里：

- 勾选/取消 `锁定启用`
- 在 `目标输入法` 列表里选择要锁定的输入源（建议选「微信输入法」对应的那一项）

## 登录时启动

菜单栏里提供 `登录时启动` 开关（Launch at Login）。

- `登录时启动` 需要 macOS 13+（更旧系统会显示“系统不支持”）
- 建议将 App 放到 `/Applications` 后再开启，否则系统可能显示为“未找到/需批准”
- 这里指“用户登录后自动运行”；如果你需要“未登录就随系统启动”，需要 LaunchDaemon（管理员权限），不属于本工具范围
