# 输入法锁定

`输入法锁定`（`IMELocker`）是一个轻量的 macOS 菜单栏工具，用来把系统输入法稳定锁定到你指定的目标输入法。

它适合这种场景：你明明已经切到微信输入法、搜狗或别的中文输入法，但某些应用会突然把输入法切回 `ABC`、英文或别的输入源，而且还不容易切回来。这个工具会在检测到切换后，自动把输入法拉回你选定的目标输入法。

除了监听系统输入法变化通知，它还会做周期性检查，所以在通知没有及时触发的情况下，也能兜底恢复。

## 功能特性

- 锁定到指定输入法，例如微信输入法
- 菜单栏快速开关锁定状态
- 自动检测并切回目标输入法
- 支持登录时启动
- 提供重启菜单项，方便刷新运行状态
- 检测到 `Secure Input` 时自动暂停强制切换，避免密码框场景反复抢输入法

## 系统要求

- macOS 12 或更高版本
- 建议 macOS 13+ 以获得更稳定的“登录时启动”体验

## 下载与安装

发布版本在 GitHub Releases：

[Releases](https://github.com/Forest227/IMELocker/releases)

安装步骤：

1. 下载最新的 `.dmg` 安装包。
2. 打开后把 `输入法锁定.app` 拖到 `Applications`。
3. 首次启动时如果被系统拦截，在“系统设置 → 隐私与安全性”中选择仍要打开。

注意：

- 当前发布版本未做开发者签名和 Apple 公证，首次打开可能会看到 macOS 安全提示
- 建议把应用放到 `Applications` 后再开启“登录时启动”

## 使用方法

启动后，应用会出现在菜单栏中。

首次使用建议：

1. 打开菜单栏图标。
2. 确认 `锁定启用` 已打开。
3. 在 `目标输入法` 中选择你想锁定的输入法。
4. 如果需要开机自动运行，再打开 `登录时启动`。

菜单栏中目前包含这些常用操作：

- `锁定启用`
- `目标输入法`
- `登录时启动`
- `重启`
- `退出`

## 工作原理

应用会通过 macOS 输入源相关 API 获取当前输入法，并在以下时机尝试恢复目标输入法：

- 系统输入法切换通知到来时
- 前台应用切换时
- 周期性检查触发时

如果当前处于 `Secure Input` 场景，例如密码输入框、部分系统授权窗口或某些安全输入环境，应用会暂停强制切换，避免反复抢输入法。

## 已知限制

- macOS 在密码框等 `Secure Input` 场景下可能禁止第三方输入法工作；本工具会在这类场景暂停强制切换
- 某些应用会主动调用系统 API 改写输入法，本工具只能在切换发生后再尝试切回
- “登录时启动”依赖系统登录项能力，旧系统上可能不可用或需要用户额外批准
- 当前仓库提供的是未签名版本，企业环境或更严格的安全策略下可能需要手动放行

## 从源码构建

在仓库根目录执行：

```bash
./build.sh
```

生成产物：

```bash
build/输入法锁定.app
```

## 打包 DMG

本地生成安装包：

```bash
./scripts/build_dmg.sh
```

默认输出：

```bash
build/IMELocker-v0.0.1.dmg
```

也可以指定版本号：

```bash
./scripts/build_dmg.sh 0.0.1
```
